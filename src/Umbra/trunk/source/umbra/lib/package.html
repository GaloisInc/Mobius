This package contains the classes which handle various helper tasks. In particular it contains all the exceptions defined in the plugin. The package handles the following taks:
  • the operations on history {@see HistoryOperations}
  • the operations of file names {@see FileNames}
  • the constants with GUI messages {@see GUIMessages}
  • the constants with Eclipse identifiers {@see EclipseIdentifiers}
  • the connection with BMLLib {@see BMLParsing}
  • {@see BytecodeStrings} together with its superclasses {@see BytecodeStringsMnemonics} and {@see BytecodeStringsGeneric} contain the strings of different keywords used in the byte code documents.
  • exceptions
    • {@see UmbraException} - generic exception used to signal various situations, this should always be caught within Umbra.
    • {@see UmbraRuntimeException} - generic runtime exception used to signal various situations, this should always be caught within Umbra.
    • {@see UmbraLocationException} - an exception used to signal a situation in which the processing of a byte code document went behind the ranges of the current document.
    • {@see UmbraMethodException} - an exception used to signal a situation in which a method number is referenced which exceeds the number of methods in the byte code class.

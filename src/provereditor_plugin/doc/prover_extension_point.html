Prover

Identifier:
prover.editor.prover

Since:
The first release.

Description:
This extension point is used to add a new prover to the ProverEditor.

Configuration Markup:

<!ELEMENT extension (language)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED>


<!ELEMENT language EMPTY>

<!ATTLIST language

name       CDATA #REQUIRED

extension  CDATA #REQUIRED

translator CDATA #REQUIRED

toplevel   CDATA #REQUIRED>

  • name - The name by which the prover will be called afterward.
  • extension - The extension of the files used for this prover.
  • translator - The class used to mainly to parse the different files.
  • toplevel - The class to help interact with the toplevel.

Examples:

An example which is used with the Coq prover: 

<extension point=
"prover.editor.prover"
>
 
       
<language toplevel=
"fr.inria.everest.coq.editor.BasicCoqTop"

                          extension=
".v"

                          name=
"Coq"

                          translator=
"fr.inria.everest.coq.editor.CoqProverTranslator"
/>


</extension>

API Information:
The translator attribute specified class must extend the class prover.plugins.AProverTranslator. The toplevel attribute specified class must extend the class prover.plugins.IProverTopLevel.

Supplied Implementation:
No.

The package represents the BML attributes and provides procedures to load and save them to class files. The top-level package contains classes and interfaces which represent generic functionality of attributes. We have here:
  • {@see AttributeNames} - the class with string representations of all the attribute names used in the project
  • {@see AType} - a container for flags that allow to retrieve different kinds of attributes from their containers (e.g. class, method or field)
  • {@see BCAttributeTable} - a representation of tables of attributes
  • {@see BCPrintableAttribute} - an abstract class with methods to parse and print attributes
  • {@see IBCAttribute} - an interface which should be implemented by all BML attributes

This package contains subpackages which divide the space of existing attributes based on their location within a class.

The subpackages of annot.attributes are:

  • {@see annot.attributes.clazz} - attributes located in the attribute area for a class
  • {@see annot.attributes.field} - attributes located in the attribute area within a field representation
  • {@see annot.attributes.method} - attributes located in the attribute area within a method representation

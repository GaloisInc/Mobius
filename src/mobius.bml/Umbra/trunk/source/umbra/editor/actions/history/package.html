This package contains all the classes which handle the GUI actions available in the bytecode Umbra editor and which handle the access to the historical versions of a bytecode file. The package handles the actions:
  • To add an item to the history {@see HistoryAction}
  • To get an item from the history {@see BytecodeRestoreAction}
  • To clear the history concerning the currently edited file {@see ClearHistoryAction}
The history is organised in files. In case we add an item to the history, the current file name.btc is copied to name.bt<num> where <num> is the number of the first free history item.

The package represents the BML attributes that are located in the method attribute areas. It provides procedures to load and save them to class files.

All the BML attributes of a method are held in a single container class {@link annot.bcclass.BCAttributeMap}.

We have two kinds of method attributes:

  1. Table-like attributes: asserts, loop specifications. These attributes are represented by subclasses of the abstract class {@link BCAttributeTable}
  2. Attributes for single clauses: method specification. These attributes are represented by subclasses of the abstract class {@link MethodAttribute}

The table-like attributes are currently:

  1. {@link AssertTable} - which gathers information on all the asserts in the body of a method
  2. {@link LoopSpecificationTable} - which gathers information on all loop specifications

The table-like attributes contain annotations associated with the instructions of the method. These annotations are represented by subclasses of {@link InCodeAnnotation}. The subclasses of the {@link InCodeAnnotation} are:

  1. {@link SingleAssert} - which represents assert clauses
  2. {@link SingleLoopSpecification} - which represents loop specification sets that contain: invariant and variant

The attributes for single clauses are currently:

  1. {@link MethodSpecification} - which gathers information on the whole method header specification in a simplified view this is a sequence of (generalised) requires-ensures pairs

A single method header with BML specifications is represented by {@link MethodSpecificationAttribute}. This attribute contains a sequence of specification cases (roughly speaking requires-ensures pairs) represented by objects of {@link SpecificationCase}.

Each {@link SpecificationCase} object consists of (possibly null):

  1. expression for precondition
  2. expression for postcondition
  3. modifies expression
  4. a (possibly empty) sequence of exceptional conditions

This package contains also an auxiliary class {@link SingleList} which is used to gather together all the asserts associated with a single instruction in the bytecode.

State Diagrams and Navigation Graphs

An alternative way to alleviate the burden of annotating code is to provide some higher-level specification formalism, which makes developing specifications less work. These high-level specifications are then automatically translated into to more basic program annotations.

The earlier work on security requirements reported in Deliverable 1.2 [mC06a] suggested that so-called midlet navigation graphs [Cré06] could be such a higher-level specification formalism. Such graphs are essentially a form of security automata. The NAVMID tool developed at France Télécom provides support in drawing midlet navigation graphs. The existing industry standard for testing MIDP application, the Unified Testing Criteria (UTC) [Ini06] already prescribes the use of a similar, albeit completely informal, notion of `flow diagram.' Unfortunately, the notion of flow diagrams is completely informal 6.1.
